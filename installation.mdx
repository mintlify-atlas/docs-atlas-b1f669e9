## Prerequisites

To install pwgrs, you'll need:

- Rust toolchain (rustc and cargo) version 1.56 or later
- A Unix-like operating system (Linux, macOS) or Windows

<Note>
  If you don't have Rust installed, visit [rustup.rs](https://rustup.rs/) to install the Rust toolchain.
</Note>

## Install from crates.io

The easiest way to install pwgrs is using cargo:

<Steps>
  <Step title="Install with cargo">
    Open your terminal and run:
    
    ```bash
    cargo install pwgrs
    ```
    
    This will download, compile, and install pwgrs from the official Rust package registry.
  </Step>
  
  <Step title="Verify installation">
    Check that pwgrs is installed correctly:
    
    ```bash
    pwgrs --version
    ```
    
    You should see output like:
    
    ```
    pwgrs 1.3.2
    ```
  </Step>
  
  <Step title="Generate your first password">
    Test the installation by generating a password:
    
    ```bash
    pwgrs
    ```
    
    The password will be automatically copied to your clipboard and printed to the terminal.
  </Step>
</Steps>

## Build from source

If you want to build pwgrs from source or contribute to development:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/box/pwgrs.git
    cd pwgrs
    ```
  </Step>
  
  <Step title="Build the project">
    Build pwgrs in release mode for optimal performance:
    
    ```bash
    cargo build --release
    ```
    
    The compiled binary will be located at `target/release/pwgrs`.
  </Step>
  
  <Step title="Install locally">
    Install the locally built version:
    
    ```bash
    cargo install --path .
    ```
    
    This will install pwgrs to your cargo bin directory (typically `~/.cargo/bin/`).
  </Step>
  
  <Step title="Run tests">
    Verify everything works by running the test suite:
    
    ```bash
    cargo test
    ```
  </Step>
</Steps>

## System-specific installation

### Arch Linux (AUR)

For Arch Linux users, pwgrs can be installed from the AUR using the included PKGBUILD:

```bash
# Using an AUR helper like yay
yay -S pwgrs

# Or manually with makepkg
cd /path/to/pwgrs
makepkg -si
```

### Add to PATH (if needed)

If the `pwgrs` command is not found after installation, ensure your cargo bin directory is in your PATH:

<CodeGroup>
```bash bash/zsh
echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

```bash fish
fish_add_path $HOME/.cargo/bin
```

```powershell Windows PowerShell
$env:Path += ";$env:USERPROFILE\.cargo\bin"
```
</CodeGroup>

## Dependencies

pwgrs has minimal dependencies, all managed by cargo:

- **rand** (0.9.1): Core random number generation traits
- **rand_chacha** (0.9.0): ChaCha20 RNG implementation
- **clap** (4.5.7): Command-line argument parsing
- **arboard** (3.4.0): Cross-platform clipboard support

All dependencies are automatically installed when you install or build pwgrs.

## Updating pwgrs

To update to the latest version:

```bash
cargo install pwgrs --force
```

The `--force` flag tells cargo to reinstall even if pwgrs is already installed.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command not found after installation">
    Make sure `~/.cargo/bin` is in your PATH. Add it to your shell configuration file:
    
    ```bash
    export PATH="$HOME/.cargo/bin:$PATH"
    ```
  </Accordion>
  
  <Accordion title="Clipboard not working">
    On Linux, clipboard support requires X11 or Wayland display server. If running on a headless server, the clipboard feature won't work, but password generation will still function normally.
  </Accordion>
  
  <Accordion title="Build fails on Windows">
    Ensure you have the Microsoft C++ Build Tools installed. The Rust toolchain needs a linker, which is provided by Visual Studio or the standalone Build Tools.
  </Accordion>
</AccordionGroup>
