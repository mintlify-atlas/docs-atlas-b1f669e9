## Your first password

Generating a password with pwgrs is as simple as running the command:

```bash
pwgrs
```

This generates a password with all character sets (lowercase, uppercase, numbers, and special characters) with enough length to guarantee at least 256 bits of entropy. The password is automatically copied to your clipboard.

<Note>
  With all character sets enabled (90 distinct characters), the default length is 40 characters to achieve 256 bits of entropy.
</Note>

## Basic usage

### Generate with specific character sets

Control which types of characters appear in your password:

<CodeGroup>
```bash Lowercase only
pwgrs -a
# Example: aachgovgbekfrgebxfscruwjgrmzzxsicakzwqtlcrwarpqntkpvydk
# Length: 55 (26 distinct chars)
```

```bash Uppercase only
pwgrs -A
# Example: YPFNCITWYLOGEEIPLWUWIOUJDTIVJFRTXQSDAORFOGXFUHZFLDYXKZY
# Length: 55 (26 distinct chars)
```

```bash Mixed case
pwgrs -Aa
# Example: ugSVnQabbnkKhxdCrhleBCyUnlLqUojUMVTUnJSfMHTTJ
# Length: 45 (52 distinct chars)
```

```bash Numbers only
pwgrs -n
# Example: 404033964956362206872986540699097332184979913456121844514705164504017662414846
# Length: 78 (10 distinct chars)
```

```bash Special characters
pwgrs -s
# Example: )<:}=_)=/{@!:/[/[.\])-^~</<,)>.\,}=^*(&}([&#_,=&^\&|(=
# Length: 54 (29 distinct chars)
```
</CodeGroup>

<Info>
  The length automatically adjusts based on your selected character sets to maintain 256 bits of entropy. Fewer character types mean longer passwords.
</Info>

### Set a custom length

Override the automatic length calculation:

```bash
pwgrs -l 20
# Example: ?vi_H1tH6_F}1a3.5lnO
```

<Warning>
  Setting a custom length may result in less than 256 bits of entropy. Only do this if you understand the security implications.
</Warning>

### Generate multiple passwords

Create several passwords at once:

```bash
pwgrs -l 20 -c 5
```

```
33gM6q_NY+)QmFu?hSon
1!sF~127(&Sb[2b)c/J}
9hi0yjw_bGOF92LIph=0
#1mMVsk5I7e&sfg>p9U0
M5WdLId/00!}h=v741mN
```

<Note>
  When generating multiple passwords, they are **not** copied to the clipboard automatically.
</Note>

## Special commands

pwgrs includes specialized subcommands for common use cases:

### Secret generation

Generate secrets suitable for API keys and tokens (uses only alphanumeric characters):

```bash
pwgrs secret
# Example: o93Zwf73lNphrq997mt76j0c5M1P7IQnyG86rDCBf98
```

The `secret` command:
- Uses lowercase letters, uppercase letters, and numbers (no special characters)
- Guarantees 256 bits of entropy
- Automatically copies to clipboard

### WiFi passwords

Generate WiFi-friendly passwords with convenient formatting:

```bash
pwgrs wifi
# Example: 2fad-03e3-930h-ko1s
```

The `wifi` command:
- Uses only lowercase letters and numbers (no confusing characters)
- Provides 128 bits of entropy (sufficient for WPA2/WPA3)
- Formats with hyphens for easy typing
- Automatically copies to clipboard

Generate multiple WiFi passwords:

```bash
pwgrs wifi -c 3
```

## Command reference

Here's a quick reference of all available options:

| Flag | Long form | Description |
|------|-----------|-------------|
| `-l` | `--length` | Set password length (overrides automatic calculation) |
| `-c` | `--count` | Generate multiple passwords |
| `-a` | `--alphabet` | Include lowercase letters (a-z) |
| `-A` | `--alphabet-uppercase` | Include uppercase letters (A-Z) |
| `-n` | `--numbers` | Include numbers (0-9) |
| `-s` | `--special` | Include special characters (!, @, #, etc.) |
| `-h` | `--help` | Show help information |
| `-V` | `--version` | Show version information |

### Subcommands

| Command | Description |
|---------|-------------|
| `secret` | Generate alphanumeric secrets with 256-bit entropy |
| `wifi` | Generate WiFi-friendly passwords with 128-bit entropy |

## Understanding entropy

pwgrs automatically calculates the password length needed for 256 bits of entropy using the formula:

```
length = ⌈(256 × log(2)) / log(distinct_chars)⌉
```

For example:
- **90 characters** (all sets): 40-character password
- **62 characters** (alphanumeric): 43-character password  
- **52 characters** (mixed case): 45-character password
- **26 characters** (single case): 55-character password
- **10 characters** (numbers only): 78-character password

<Info>
  256 bits of entropy is considered secure against all known attacks, including quantum computers for the foreseeable future.
</Info>

## Using as a library

You can integrate pwgrs into your Rust projects:

<Steps>
  <Step title="Add dependency">
    Add pwgrs to your `Cargo.toml`:
    
    ```toml
    [dependencies]
    pwgrs = "1.3.2"
    ```
  </Step>
  
  <Step title="Import and use">
    ```rust
    use pwgrs::*;
    
    fn main() {
        // Use default character sets
        let charsets = vec![
            Charset::new(&CHARSET_ALPHABET),
            Charset::new(&CHARSET_ALPHABET_UPPERCASE),
            Charset::new(&CHARSET_NUMBERS),
            Charset::new(&CHARSET_SPECIAL),
        ];
        
        // Calculate length for 256-bit entropy
        let char_count = count_chars_in_charsets(&charsets);
        let length = get_length_for_entropy(256, char_count);
        
        let options = GeneratorOptions { length, charsets };
        let password = gen_password(&options);
        
        println!("Generated password: {}", password);
    }
    ```
  </Step>
</Steps>
